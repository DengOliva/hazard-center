# 隐患数据中心

统一的数据文件入口。网页会先根据文件名识别数据类型，再交给对应模块更新。当前已支持 `安全隐患信息表_XXXXXX.xlsx` → “隐患录入统计”，并支持隐患查询、人员分类与标准维护、任意时间区间达标统计，以及按“检查单位”比较中建二局和外部单位的录入数量。

## 本地运行

```powershell
python -m pip install -r requirements.txt
python app.py
```

浏览器访问 <http://127.0.0.1:8010>。首次启动会自动导入 `seed` 中的 2026-07-02 数据，并从旧统计工具读取人员和标准。

## 统计口径

- 人员录入数：`检查人姓名` 去掉开头的人员编号后，按姓名精确匹配。
- 单位对比：`检查单位 = 中建二局` 为内部，其余（含空值）为外部。
- 每日标准：标准数量 × 所选区间自然日数。
- 每周标准：标准数量 × `向上取整(区间自然日数 / 7)`。
- Excel 更新采用完整快照：新文件导入成功后替换旧隐患列表，不累计重复记录。

## Docker

```bash
docker compose up -d --build
```

服务只映射至服务器本机 `127.0.0.1:8010`，上线时由 Nginx/Caddy 将独立子域名反向代理至该端口。`data/` 挂载在宿主机，包含 SQLite 数据库和上传文件，部署前应配置定期备份。
